[Reading Notes(CN注意)]Label Augmentation for Zero-Shot Hierarchical Text Classification

DizzyHardSugar
2024-12-09

创新点：

  1. 提出一种新的标签增强技术，能够为深层级标签生成新的子标签。
  2. 扩展了 UP 方法，使其能改进叶节点的分类性能。
  3. 提出了一组指标用于评估层级标签的密度和细粒度。
  4. 在四个公开数据集上进行了实验，在三个数据集上实现了最新的 SOTA（state-of-the-art）结果。

Method

基于Bongiovanni 等人（2023）提出的上推分数传播（Upward Score Propagation, UP）方法。
在层级$H$，文档 $d$ 和标签 $c_{j}^{l}$​分别映射到相同的语义向量空间，表示为 $\Psi_d$​ 和 $\Psi_c$​。初始相关性分数为$$p(c_{j}^{l}) = SC(\Psi_d(d), \Psi_c(c_{j}^{l}))$$其中SC是余弦相似度。对于一个层级$l$中的所有标签，都会计算与文档的相关性分数 $p(c_{j}^{l})$。
随后，作者定义了 UP 方法，该方法通过层级结构向上传播置信分数，将初始相关性分数更新为后验分数 $S_{UP}(c_{j}^{l})$。其基本理念是：如果某个标签与文档相关，那么它的父节点也应该相关。

符号 含义
$l$ 层级的深度，$l = 0, \dots, L$。
$c_{j}^{l}$ 层级 $l$ 中第 $j$ 个标签。
$N_l$ 层级 $l$ 中标签的数量。
$c_0$ 层级的根节点，通常是无名标签或数据集的名称。
$\uparrow c_{j}^{l}$ 标签 $c_{j}^{l}$ 的父类别（上一层的标签）。
$\Uparrow c_{j}^{l}$ 标签 $c_{j}^{l}$​ 的所有祖先类别（从根节点到其父节点的所有路径上的标签）。
$\downarrow c_{j}^{l}$ 标签 $c_{j}^{l}$ 的直接子类别（下一层的标签）。
$\Downarrow c_{j}^{l}$ 标签 $c_{j}^{l}$ 的所有后代类别（从其子节点到叶节点的所有路径上的标签）。
$(\downarrow c_{j}^{l})_i$ 标签 $c_{j}^{l}$​ 的第 iii 个子类别。
对于深度为 $L$ 的标签，其 UP 分数直接等于初始分数：$S_{UP}(c_{j}^{L}) = p(c_{j}^{L})$。对于层级中小于 $L$ 的某一层级 $l$，其标签的分数则是递归定义的，需要引入以下符号：
  • n：标签$c_{j}^{l}$的子类别数量（即 $n = |\downarrow c_{j}^{l}|$）。
  • $S_{UP}^{(i)}(c_{j}^{l})$：在第 iii 个子类别的基础上更新的分数，具体公式为：

$$S_{UP}^{(i)}(c_{j}^{l}) = \begin{cases} \max(p(c_{j}^{l}), 0), & \text{若 } i = 0, \ S_{UP}^{(i-1)}(c_{j}^{l}), & \text{若 } (\downarrow c_{j}^{l})i \text{ 不相关(第i个孩子分数小于第i-1次更新后分数)}, \ S{UP}^{(i-1)}(c_{j}^{l}) \cdot e^{\delta_{j,i}}, & \text{若 } (\downarrow c_{j}^{l})i \text{ 相关(目前分数小于第i个孩子上推分数)}, \ S{UP}((\downarrow c_{j}^{l})_i), & \text{若子类别分数更高(第i个孩子的上推分数大于超参数/门槛值)}. \end{cases}$$
note：

  • $S_{UP}^{(i)}(c_{j}^{l})$表示当前节点$c_{j}^{l}$的 第 i 次更新后的上推分数（UP score），通过迭代地结合 $c_{j}^{l}$的子节点$(\downarrow c_{j}^{l})i$的分数计算出来的。每次更新都会逐步整合更多子节点的信息，最终在第 $n = |\downarrow c{j}^{l}|$次迭代后，得到完整的 $S_{UP}(c_{j}^{l})$。
  • $S_{UP}((\downarrow c_{j}^{l})i)$是子节点 $(\downarrow c{j}^{l})i$的最终上推分数，它是通过对子节点自身的分数进行递归计算得到的，而不是直接与 $c{j}^{l}$ 的分数相关。子节点的分数通常取决于其下级节点（如果有下级节点）或者它的初始分数 $p((\downarrow c_{j}^{l})_i)$。
  • 第一行是截断项更新，第四行是类似DP的第一个值更新；第二行表示第i个孩子分数不值得更新当前分数，第三行是第i个孩子的上推分数可以更新当前分数。

标签增强：

假设有一个数据集 $D$，其标签被组织成深度为 $L$ 的层级 $H$。方法使用一个预训练的大语言模型（LLM），通过为每个叶节点 $c_{j}^{L}$添加新叶节点集合 $\downarrow c_{j}^{L}$ 来增强标签层级。新生成的标签与原始层级结构保持一致，并比 $c_{j}^{L}$​ 更具体。我们假设 $H$ 的每个节点只有一个父节点，因此层级可以用树表示。

通过提示（prompt）LLM 来生成 $\downarrow c_{j}^{L}$，提示信息由层级结构中提取的上下文组成。原则上，我们希望将整个层级结构包含在提示中，并让 LLM 为所有 $N_{L}$ 个叶节点生成新的子标签。然而，这种方法在许多情况下不可行，原因包括：1）提示可能超出 LLM 的单次处理能力；2）输出可能过长或复杂，无法可靠生成。

因此，提出了一种迭代方法：为每个 $L-1$ 层级节点的所有子节点同时生成扩展层级。具体而言，定义节点 $c_{j}^{L-1}$ 的分支集 $B_{j}^{L-1}$​ 为包含以下所有标签的集合：
$$B_{j}^{L-1} = c_{j}^{L-1} \cup ⇑ c_{j}^{L-1} \cup ↓ c_{j}^{L-1}$$
然后，为每个分支集创建通用提示模板，形式如下：

$$[c_{j}^{L-1}] [⇑ c_{j}^{L-1}] 对象可以分类为 [↓ c_{j}^{L-1}]。您能否为这些类别提供一些更具体的分类？$$

例如，在一个产品评论数据集中，如果 $L-1$ 层级有一个标签 “skin care”，其父标签为 “beauty”，子标签为 “face”、”body” 和 “sun”，那么分支集将包含 “skin care”、”beauty”、”face”、”body” 和 “sun”。相应的提示为：

“skin care” “beauty” 产品可以分类为 “face”、”body” 或 “sun”。您能否为这些类别提供一些更具体的分类？

生成的新标签层级可以与 UP 方法结合使用，作为深层级 HTC 的基础。需要注意的是，生成的新标签并非最终的分类目标，而是为分类步骤提供更多上下文信息，从而提高预测性能。
$$
\textbf{Algorithm: The HiLA Algorithm} \
\textbf{Input:} \text{Labels hierarchy } H \
\textbf{Output:} \text{H is extended with a new leaf level} \
\text{for } j \in N_{L-1} \text{ do} \
\quad B_{j}^{L-1} \gets c_{j}^{L-1} \cup \Uparrow c_{j}^{L-1} \cup \downarrow c_{j}^{L-1} \
\quad P \gets \text{fill_template}(B_{j}^{L-1}) \
\quad \downarrow c_{j}^{L} \gets \text{parse}(\text{LLM}(P)) \
\text{end}
$$

簇密度估计:

方法高度依赖于层级结构的质量。由于新标签的生成依赖于现有标签，标签嵌入的密度成为生成质量的关键决定因素。更高的密度意味着标签空间更语义丰富、组织良好，从而增强增强过程的有效性。

定义了两种基于余弦相似度的指标：

  1. D1 指标：衡量某个标签的子标签之间的平均相似度，用于评估子标签的紧密程度。
  2. D2 指标：衡量父标签与其所有子标签的平均相似度，用于评估子标签与父标签的关联性。

簇的定义

我们定义一个标签簇（label cluster，简称簇）为以下集合：

$$C_{j}^{l+1} = c_{j}^{l} \cup \downarrow c_{j}^{l},$$

即一个标签 $c_{j}^{l}$ 和它的所有子标签的集合。

D1 指标

D1 用于衡量某个标签的子标签之间的平均相似度，定义如下：
$$D1(C_{j}^{l+1}) = \frac{\sum_{i,k} SC(\Psi_c((\downarrow c_{j}^{l})i), \Psi_c((\downarrow c{j}^{l})k))}{\binom{|\downarrow c{j}^{l}|}{2}},$$

其中：

  • $SC$ 表示余弦相似度。
  • $(\downarrow c_{j}^{l})i$和 $(\downarrow c{j}^{l})k$ 是标签 $c{j}^{l}$​ 的第 $i$ 和第 $k$ 个子标签。
  • $|\downarrow c_{j}^{l}|$ 是标签 $c_{j}^{l}$ 的子标签数量。
  • 分母 $\binom{|\downarrow c_{j}^{l}|}{2}$ 表示标签子节点中可能的两两组合数。

作用：D1 测量标签的子标签之间的紧密程度，子标签之间的相似度越高，D1 值越大。

D2 指标

D2 用于衡量某个标签与其子标签之间的平均相似度，定义如下：
$$D2(C_{j}^{l+1}) = \frac{\sum_{i=1}^{|\downarrow c_{j}^{l}|} SC(\Psi_c(c_{j}^{l}), \Psi_c((\downarrow c_{j}^{l})i))}{|\downarrow c{j}^{l}|},$$

其中：

  • $SC$ 表示余弦相似度。
  • $c_{j}^{l}$​ 是当前标签节点。
  • $(\downarrow c_{j}^{l})i$是标签 $c{j}^{l}$的第 $i$ 个子标签。
  • 分母 $|\downarrow c_{j}^{l}|$ 是当前标签的子标签数量。

作用：D2 测量标签与其子标签之间的关联性，子标签越接近父标签，D2 值越大。

层级整体计算

将上述两个指标应用到层级结构中的每一层，方法如下：

  • 对每个层级 $l’ > 0$，我们对所有属于该层的簇 $C_{j}^{l’}$ 的指标值取平均值。
  • 需要注意的是，由于根节点 $c_0$​ 没有嵌入表示，D2 在层级 $l’ = 1$ 时无定义。

Results

  • 实验数据集
    作者在以下四个公开的层级文本分类（HTC）数据集上验证了方法的有效性：
    • DBPedia Classes: 包括 Wikipedia 文章，层级深度为 3。
    • Web Of Science: 包括科学领域的研究摘要，层级深度为 2。
    • Amazon Product Reviews: 包括产品评论，层级深度为 3。
    • Books Blurbs: 包括书籍简介，层级深度为 2。
  • 主要实验结果
    • 提出的 HiLA + UP 方法 在三个数据集（DBPedia、Amazon、Books）上达到了 新的最先进（SOTA）性能，特别是在最深层级（叶节点）上效果显著。
    • 对比结果表明，结合 HiLA 进行标签增强后的性能优于仅使用 UP 方法或传统 ZSC 方法（基于嵌入相似度）。
    • 在 Web Of Science 数据集上，HiLA 方法未带来提升，且分类性能略有下降。
  • 指标分析
    • 使用余弦相似度指标 D1 和 D2 测量层级的密度与一致性：
      • 高 D1 和 D2 的层级更密集，表示生成的新标签具有更高的语义一致性。
      • Web Of Science 数据集中，标签的一致性随着层级深度增加而下降，表明标签生成不够具体，导致分类性能下降。
    • 实验证实 D1 和 D2 与分类性能高度相关，可作为判断层级质量的前期工具。

结论

  1. 主要贡献：
    • 标签增强方法（HiLA） 提高了层级标签的粒度与语义密度，使分类模型在深层级（尤其是叶节点）上的表现大幅提升。
    • 新指标 D1D1D1 和 D2D2D2 可以定量分析层级结构的语义质量，帮助评估 HiLA 方法的适用性。
  2. 实验验证：
    • 在四个数据集上进行了全面验证，其中三个数据集表现优异，证明方法的通用性和有效性。
    • HiLA 方法生成的标签不仅与其父节点一致，还与整个分支语义相关性较高，从而提升了分类性能。
  3. 方法价值：
    • HiLA 提供了一种在零样本环境下处理层级文本分类的新途径，适合需要动态扩展标签体系的应用场景（如电子商务、文档组织等）。

Weakness

  1. 依赖层级输入质量：
    • HiLA 方法依赖于输入的标签层级是否语义清晰且密集。如果初始层级较差（如 Web Of Science 数据集的标签一致性较低），生成的子标签可能无效或过于通用，反而降低分类性能。
  2. 标签生成质量限制：
    • 标签增强依赖于预训练的大语言模型（LLM）。如果 LLM 对领域知识的掌握不足，可能会生成语义无关或冗余的标签。
    • 未控制生成标签的数量和质量，可能导致层级结构过度复杂。
  3. 工具依赖：
    • 研究中使用了 OpenAI 的 GPT-3.5 模型，这是一种闭源工具，限制了方法的可复现性和对开源系统的推广。
    • 如果将方法迁移到其他 LLM（如 LLaMA 或 Orca-2），效果尚未验证。
  4. 数据集适应性有限：
    • 对于层级深度较小、标签多样性不足的数据集（如 Web Of Science），方法的优势不明显，可能无法有效提升性能。
